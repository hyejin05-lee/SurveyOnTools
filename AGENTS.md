# AGENTS.md — Coding Agents 비교 작업 규칙

이 저장소는 유명/특수목적 **Coding Agents 기능 비교표**를 만드는 협업 프로젝트다.
여러 명의 사용자와 자동화된 Agent가 **동시에** 작업하며, 각자의 산출물을 취합해 최종
report로 합친다. 모든 Agent는 작업 전 이 파일과 `GOAL.md`를 먼저 읽는다.

## 1. 목표 (요약)
- 대상 도구들이 제공하는 **고급(advanced) 기능**의 비교표를 Markdown으로 작성한다.
- 산출물은 ① 지원 기능 목록, ② 도구별 지원 여부/레벨, ③ 향후 유사 도구 개발 아이디어를 담는다.
- 전체 목표 원문은 `GOAL.md` 참조. 충돌 시 `GOAL.md`가 우선한다.

## 2. 비교 대상 도구 (7종, 고정)
| 분류 | 도구 |
|------|------|
| 범용 Coding Agents | Claude Code, Codex CLI, Antigravity, Cline |
| 특수 목적 Agents | OpenClaw, oh-my-openagent, Hermes Agent |

## 3. 다룰 기능 / 제외할 기능
- **포함(핵심):** Agentic Memory, Harness Engineering, Context Compaction(토큰 절감),
  Adaptive/개인화, 도구 연결성, 멀티/서브 에이전트 오케스트레이션, Skills/확장,
  멀티모델 라우팅 등 차세대 설계에 영향을 주는 고급 기능.
- **제외:** 단순 코드 자동완성, 기본 파일 편집 등 가장 간단한 기능은 모두 무시한다.

## 4. 산출물 형식 규칙
1. **최상단에 "한눈에 보는 비교표"** 를 둔다 — 한 화면에서 파악 가능해야 한다(기능 행 ×
   도구 열). 부가 설명은 표 **아래에 분리**한다.
2. 지원 레벨 범례는 다음으로 통일한다:
   **● 1급(native·성숙) · ◐ 부분 지원(제한적·간접) · ○ 미지원/미확인**
3. 본문 각 주장에는 `[n]` 형태의 인용 번호를 달고, 문서 하단 **References**에 확인 가능한
   URL을 매핑한다. 사용한 인용 번호와 References 목록은 반드시 일치시킨다(유령/누락 금지).
4. 마지막에 **향후 유사 도구 개발 아이디어** 섹션을 포함한다.
5. 외부 참조 문헌은 **최대한 많이** 활용한다. 1차 출처(공식 docs·GitHub·논문)를 우선하고
   2차 출처(리뷰·블로그)는 보조로 표기한다.

## 5. 파일 / 폴더 규칙
- 각 Agent/사용자는 **자신의 작업 폴더** 아래에 산출물을 만든다(예: `claude/`, `gemini/`, `antigravity`).
  남의 폴더나 루트 파일을 수정하지 않는다(공유 파일 충돌 방지).
- 파일명은 작업자가 결정 가능하되 내용을 알 수 있게 짓는다
  (예: `coding-agents-feature-comparison.md`).
- 루트의 `GOAL.md`, `AGENTS.md`는 공통 규칙 파일이므로 임의 변경하지 않는다.

## 6. 작업 태도
- **특별한 경우가 아니면 사용자에게 질문하지 말고 스스로 결정**한다(`GOAL.md` 지시).
- 정보가 부족한 도구(특히 특수목적 3종)는 공개 문서가 제한적일 수 있다. 추정이 포함되면
  **문서에 명시적 caveat**을 남긴다(예: "공식 문서 제한, 추정 포함").
- 날짜는 **절대 표기**로 적는다(예: "2026-06-17"). "오늘/최근" 같은 상대 표현 지양.
- 작성 기준일을 문서에 명시해 취합 시 시점을 알 수 있게 한다.
- 사실과 추정을 구분해 기술하고, 확인되지 않은 기능은 ○(미확인)으로 둔다.

## 7. 취합 친화성 (중요)
- 여러 산출물이 병합되므로, **레벨 범례·기능 명칭·도구 명칭**을 위 정의대로 통일한다.
- 표는 동일한 열 순서(Claude Code → Codex CLI → Antigravity → OpenClaw →
  oh-my-openagent → Hermes Agent)를 권장해 병합을 쉽게 한다.

## 8. 대화 기록 규칙 (`messages/` 폴더)
Agent와 사용자 간의 대화는 `messages/` 폴더에 파일로 남긴다.
- **파일명:** `NNNNNN_Writer_Title.md` — 6자리 일련번호로 시작(순서 표기),
  이어서 작성자와 짧은 제목. 예: `000001_User_Greeting.md`,
  `000002_Claude_Greeting_Reply.md`.
- **일련번호:** 6자리 0-패딩, 폴더 내 가장 큰 번호 +1로 부여한다(시간순 보장).
- **작성자(Writer):** 사람은 `User`, Agent는 자신을 식별하는 이름(예: `Claude`,
  `Gemini`)을 쓴다.
- **본문 헤더:** 파일 첫머리에 `Writer:`와 `When:`(절대 시각, KST 표기)을 적고,
  그 아래 Markdown 본문을 작성한다.
- 메시지를 받으면 새 번호로 **응답 파일을 추가**한다(기존 파일은 수정하지 않는다).
- **번호 충돌 시(병렬 작성 대비):** 같은 번호가 이미 존재하면 **자신의 파일을 다음 빈
  번호로 양보(rename)**한다. 타 작성자의 파일은 수정·이동하지 않는다. 파일명에 작성자
  식별자(`Claude`/`Gemini` 등)를 포함하므로 충돌 여부는 번호 기준으로 판단한다.
